# User Research

## Primary User Candidates

### 1. The Designer (most likely power user)
- Graphic designers, UI designers, brand designers
- Needs: HEX, RGB, CMYK, Pantone — all formats, precise values
- Workflow: sees a color IRL → needs it in Figma/Illustrator immediately
- Exports: ASE, CSS custom properties
- Pain today: takes photos, eyedroppers in Figma, loses accuracy in translation
- **Would pay for accuracy and Pantone matching**

### 2. The DIY Decorator / Home Enthusiast
- Matching paint, fabric, furniture colors
- Needs: color name, closest paint match, something they can show at a hardware store
- Doesn't care about HEX — wants "what is this color called"
- Pain today: takes photos to hardware store, uses paint brand apps that only match their own swatches
- **Shares color discoveries with friends/family**

### 3. The Fashion / Styling Person
- Matching outfit colors, building seasonal palettes
- Needs: color names, palette building, shareable results
- Emotional need: "this is my color palette" — identity and curation
- **Highly shareable — Instagram, Pinterest behavior**

### 4. The Developer
- Quickly capture colors from the physical world into code
- Needs: HEX, RGB, CSS output — fast copy-to-clipboard
- Pain today: no quick tool that goes physical → code
- **Values speed and copy precision over aesthetics**

## Recommended Primary User
**The Designer** — highest willingness to pay, clearest workflow pain, will use exports (ASE, CSS), and is the most credible early adopter to build a reputation with.

**Secondary user for v1:** Developer — overlapping needs, easy to serve with the same feature set.

## What Users Actually Want (Jobs to Be Done)
1. "I see a color in the world — I need it in my tool immediately"
2. "I want to know what this color is actually called"
3. "I want to save this for later without losing it"
4. "I want to share this color / palette with someone"
